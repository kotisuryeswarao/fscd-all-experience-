# SKILL-1: Git Version Control

## Introduction
Git Version Control manages and tracks changes in project files. It helps developers
collaborate by saving different versions of code safely.

## Tasks Completed
- ✅ STEP 1 — Initialize Git repo, configure username & email
- ✅ STEP 2 — Create files, check status, add to staging
- ✅ STEP 3 — Commit with meaningful message
- ✅ STEP 4 — Create GitHub repo & push
- ✅ STEP 5 — Create `feature-update` branch, modify, commit
- ✅ STEP 6 — Create `bug-fix` branch, modify, commit
- ✅ STEP 7 — Merge both branches into `main`
- ✅ STEP 8 — Resolve merge conflicts manually

## Setup & Usage
```bash
# Clone the repository
git clone https://github.com/YOUR_USERNAME/git-skill1.git
cd git-skill1

# Check commit history
git log --oneline --graph
```

## Branch Strategy
```
main
├── feature-update   (new features)
└── bug-fix          (bug fixes)
```
